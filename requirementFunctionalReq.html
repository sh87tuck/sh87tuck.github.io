Functional Requirements

  1. Collect direct job candidate data
    1. Job candidate enter resume data into a fillable form
    2. Job candidate submits form to system for processing
    An active job seeker is always valuable asset to a recruiting company. Any data collected from a job candidate will notice system admin.
    a Job candidate can click the link from a online job board or the recruiting company’s website which will lead the candidate to the DiscoveryCRM’s profile creating form.
    The candidate can fill out each required sections by himself or parse the resume that he already have in his storage.
    The DiscoveryCRM will show created profile and job title he or she wish to apply to review before final submit to the job.
    If a candidate is not actively looking for a job but wish to update his or her resume in DiscoveryCRM’s database,
    a job candidate may visit a recruiting company’s website and apply to the ‘any job’ opening. This will lead the candidate to profile creating UI.
    When a candidate creates his profile in the DiscoveryCRM, the system must check candidates first and last name, email, phone number and zipcode not to be blank.
    If system detect those sections are blank it will ask the candidate to fill.

    At this stage, data from the candidate’s resume transforms into tagged profile (or indexed). The tag will consists of predefined keywords and email and candidates name.
  2. Manage Data
    1. Data has been submitted to the system and is sent to Reconcile resume data process
    2. Automated duplication search is ongoing during database operation
    A candidate profile created by the job seeker will be immediately available to recruiters before it goes under reconcile process or during the reconcile process.
    Reconcile module is consisted with data search engine and A.I. that was authority to call other modules such as data extractor or data transformer.
    A.I. also can merge two profiles in the database

  3. Create New Job Candidate
    1. System determines if candidate profile exist or not
    2. System creates a new unique job candidate profile
    3. Data is loaded into the job candidate profile
    System searches database for duplication with tag (or index) if the profile is generated by candidate.
    If the profile is generated by a recruiter system only check first, last names and email address.
    The profile generated by the recruiter will be tag matched later. The original resume file is attached to profiles if available.

  4. Reconcile resume data
    1. Data is formated to fit system data specification
    2. Keyword index is checked for a match
    3. Resume data is identified as new or existing

  5. Manage job candidate profiles
    1. Placement agent add job candidate
    2. Placement agent removes job candidate
    3. Placement agent manually approves data reconciliation
